Quais construções de rede são usadas para direcionar o tráfego para o aplicativo correto em um servidor?

Quando um computador executa um serviço de rede, ele abre uma construção de rede chamada "porta" para receber a conexão. As portas são necessárias para fazer várias solicitações de rede ou ter vários serviços disponíveis.


R: ports

Quantos deles estão disponíveis em qualquer computador habilitado para rede?

Cada computador tem um total de 65535 portas disponíveis; no entanto, muitas delas são registradas como portas padrão. Por exemplo, um Webservice HTTP pode quase sempre ser encontrado na porta 80 do servidor.


R: 65535

[Pesquisa] Quantos destes são considerados "bem conhecidos"? (Estes são os números "padrão" mencionados na tarefa)

As portas "bem conhecidas" são aquelas numeradas de 0 a 1.023. Elas são atribuídas a protocolos e serviços específicos, como HTTP (porta 80) e HTTPS (porta 443), e são amplamente reconhecidas e utilizadas na comunicação de rede.


R: 1024

Primeiro, como você acessa o menu de ajuda?

O parâmetro -h mostra a página de resumo da ajuda.


R: -h

Frequentemente chamado de varredura furtiva, qual é o primeiro parâmetro listado para uma "varredura Syn"?

O parâmetro frequentemente utilizado para uma "varredura SYN", também conhecida como varredura furtiva, no Nmap é -sS.


R: -sS

Não é tão útil, mas que tal uma "varredura UDP"?

O parâmetro utilizado para realizar uma "varredura UDP" no Nmap é -sU.


R: -sU

E quanto à detecção do sistema operacional?

Ao usar -O, o Nmap tenta identificar o sistema operacional em execução no host alvo analisando características dos pacotes de resposta, como as opções TCP e os campos de TTL (Time to Live).


R: -O

E quanto à detecção da versão do serviço?

Com o parâmetro -sV, o Nmap identifica não apenas quais portas estão abertas, mas também tenta determinar a versão dos serviços em execução nessas portas.


R: -sV

A maioria das pessoas gosta de ver alguma saída para saber se a varredura está realmente fazendo alguma coisa. Qual é o sinalizador de verbosidade?

Ao adicionar -v ao comando, o Nmap fornece saídas mais detalhadas durante a varredura, mostrando informações sobre o progresso e as portas que estão sendo verificadas.


R: -v

Que tal "muito prolixo"? (Um favorito pessoal)

Quando você usa -vv, o Nmap fornece uma saída extremamente detalhada e abrangente


R: -vv

Às vezes, salvar a saída em um formato de documento comum pode ser muito útil para relatórios. Como salvamos a saída no formato XML?

Para salvar a saída do Nmap no formato XML, você pode usar o sinalizador -oX seguido do nome do arquivo desejado.


R: -oX

Varreduras agressivas podem ser boas quando outras varreduras não estão obtendo o resultado desejado e você realmente não se importa com o quão "barulhento" está. Qual é o botão para habilitar isso?

Para realizar uma varredura agressiva no Nmap, você pode usar o sinalizador -A.


R: -A

Como faço para definir o tempo no nível máximo, às vezes chamado de "Insano"?

Para definir o tempo no nível máximo, conhecido como "Insano" no Nmap, você pode usar o sinalizador -T5


R: -T5

E se eu quiser escanear uma porta específica?

Para escanear uma porta específica no Nmap, você pode usar o parâmetro -p seguido do número da porta que deseja verificar.


R: -p

Que tal se eu quiser escanear todas as portas?

Este comando indica ao Nmap que ele deve verificar todas as portas, de 1 a 65.535.


R: -p-

E se eu quiser habilitar usando um script do mecanismo de script nmap? Para isso, basta incluir a primeira parte do switch sem a especificação de qual script executar.

Para habilitar o uso do mecanismo de script do Nmap sem especificar um script específico, você pode usar o parâmetro --script


R: --script

E se eu quiser executar todos os scripts da categoria de vulnerabilidade?

Para executar todos os scripts da categoria de vulnerabilidade no Nmap, você pode usar o parâmetro --script seguido de vuln.


R: --script vuln

Qual opção devo incluir se não quiser fazer ping no host?

Isso informa ao Nmap para ignorar a etapa de detecção de host ativo (ping) e prosseguir diretamente para a varredura das portas, assumindo que o host está ativo.


R: -Pn

Vamos em frente e começar com o básico e executar uma varredura de sincronização na caixa fornecida. O que será esse comando sem o endereço IP do host?

Esse comando inicia uma varredura SYN, mas você precisaria adicionar o endereço IP do host ou o nome do domínio para especificar onde deseja realizar a varredura.


R: nmap -sS

Depois de escanear isso, quantas portas encontramos abertas abaixo de 1000?

Escaneie o ip da máquina do try hack me utilizando o parâmetro nmap -sS [ip_da_maquina]


R: 2
