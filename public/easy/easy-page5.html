Quais seções nos dão um controle mais intuitivo sobre nossas solicitações?

Posicionado no lado direito, o Inspector nos permite analisar e modificar solicitações de uma maneira mais intuitiva do que usar o editor raw. Exploraremos esse recurso em uma tarefa posterior.


R: Inspector

Qual visualização será preenchida ao enviar uma solicitação do módulo Proxy para o Repetidor?

Localizada no canto superior esquerdo da aba, ela exibe a lista de solicitações do Repeater. Várias solicitações podem ser gerenciadas simultaneamente, e cada nova solicitação enviada ao Repeater aparecerá aqui.


R: Request

Qual opção nos permite visualizar a página como ela apareceria em um navegador da web?

A opção render nos permite visualizar a página como ela apareceria em um navegador da web. Embora não seja comumente utilizada no Repeater, já que nosso foco geralmente está no código-fonte, ela ainda oferece um recurso valioso. Para a maioria dos cenários, a opção Pretty geralmente é suficiente. No entanto, é benéfico estar familiarizado com o uso das outras três opções.


R: Render

Qual seção no Inspetor é específica para solicitações POST?

Similar aos parâmetros de consulta, mas específico para solicitações POST. Quaisquer dados enviados como parte de uma solicitação POST serão exibidos nesta seção, permitindo-nos modificar os parâmetros antes de reenviar.


R: Body Parameters

Qual é a bandeira que você recebe?

Após capturar uma solicitação da máquina no módulo Proxy e enviar ao repeidor conseguimos ver o código-fonte da página solicitada na visualização Resposta. Usando o Inspector adicionamos um cabeçalho chamado FlagAuthorised e definimos o valor True.

Qual é o sinalizador que você recebe quando causa um erro 500 no endpoint?

Ao utilizar o burpe para fazer com que o servidor gere um código "500 Internal Server Erro" alterando o número no final da solicitação de entredas extremas, capturamos o sinalizador.
